## 1. Overview

DVH-IMS manages three categories of documents:

1. **Citizen-Uploaded Documents** — Supporting documents submitted by citizens during application
2. **Staff-Verified Documents** — Upload verification tracking by authorized staff
3. **System-Generated Documents** — Official documents generated by the system (e.g., Raadvoorstel)

## 2. Citizen Document Upload

### Bouwsubsidie — Required Documents

| Document | Code | Mandatory |
|----------|------|:---------:|
| Copy of ID | ID_COPY | ✅ |
| Inkomensverklaring (AOV/loonstrook) | INCOME_DECLARATION | ✅ |
| Land Title / Deed | LAND_TITLE | ✅ |
| Bank Statement | BANK_STATEMENT | ✅ |
| Household Composition | HOUSEHOLD_COMPOSITION | ✅ |
| CBB uittreksel / Nationaliteit verklaring | CBB_EXTRACT | ❌ |
| Gezinuittreksel | FAMILY_EXTRACT | ❌ |

### Housing Registration — Required Documents

| Document | Code | Mandatory |
|----------|------|:---------:|
| Copy of ID | ID_COPY | ✅ |
| Income Proof | INCOME_PROOF | ✅ |
| Residence Proof | RESIDENCE_PROOF | ✅ |
| Family Composition | FAMILY_COMPOSITION | ❌ |
| Medical Certificate | MEDICAL_CERTIFICATE | ❌ |
| Emergency Proof | EMERGENCY_PROOF | ❌ |

### Upload Process
1. During the wizard, the citizen sees the complete document requirements list
2. Mandatory documents are marked with an asterisk (*)
3. The citizen selects a file (PDF, JPEG, or PNG) for each requirement
4. Files are uploaded to the `citizen-uploads` storage bucket
5. Each upload is tagged with: file name, file size, timestamp, and storage path
6. The wizard prevents submission if mandatory documents are missing

### Storage
- Files are stored in Supabase Storage (`citizen-uploads` bucket)
- Access is controlled by storage policies
- Files are linked to their case/registration via the upload record

## 3. Document Verification (Staff Side)

### Process
1. Staff opens the case/registration detail view
2. Navigates to the Documents tab
3. The **Required Documents** sidebar shows all requirements with status indicators:
   - ✅ Green + asterisk (*) = Mandatory, uploaded
   - ⚪ Grey = Optional
   - ❌ Red = Mandatory, missing
4. Staff reviews each document (download and inspect)
5. Staff marks documents as **verified** when confirmed authentic

### Verification Record

| Field | Description |
|-------|------------|
| `is_verified` | Boolean — verified or not |
| `verified_by` | User ID of the verifier |
| `verified_at` | Timestamp of verification |

### Audit Trail
- Each verification action creates an `audit_event` entry
- Event includes: actor, action (DOCUMENT_VERIFIED), entity, timestamp

### Who Can Verify

| Service | Authorized Roles |
|---------|-----------------|
| Bouwsubsidie | frontdesk_bouwsubsidie, admin_staff, system_admin |
| Housing Registration | frontdesk_housing, system_admin |

---

## 4. Document Download

- Authorized staff can download uploaded documents from the case/registration detail view
- Downloads use signed URLs (time-limited, secure)
- All downloads are attributed to the requesting user's session

## 5. Generated Documents

### Raadvoorstel (Council Document)

| Aspect | Detail |
|--------|--------|
| **Service** | Bouwsubsidie only |
| **Trigger** | Automatic after Minister approval (all 8 steps completed) |
| **Format** | Generated document stored in system |
| **Storage** | `generated_document` table + file storage |
| **Access** | Available on the case detail view (Raadvoorstel tab) |
| **Audit** | Generation event is logged with actor and timestamp |

### Preconditions for Generation
- Director must have approved (Step 6)
- Ministerial Advisor must have provided recommendation (Step 7)
- Minister must have approved (Step 8)
- System blocks generation if any step is incomplete

## 6. Document Configuration

Document requirements are managed through a **shared configuration** (`src/config/documentRequirements.ts`) that serves as the single source of truth for both the public wizard and the admin document verification panel.

### Database Tables

| Table | Service | Purpose |
|-------|---------|---------|
| `subsidy_document_requirement` | Bouwsubsidie | Document type definitions with `is_active` flag |
| `subsidy_document_upload` | Bouwsubsidie | Individual upload records linked to cases |
| `housing_document_requirement` | Housing | Document type definitions |
| `housing_document_upload` | Housing | Individual upload records linked to registrations |

### Deprecated Documents (Bouwsubsidie)

Three legacy document types have been soft-deprecated (`is_active = false`):
- Building Permit (BUILDING_PERMIT)
- Construction Plan (CONSTRUCTION_PLAN)
- Cost Estimate (COST_ESTIMATE)

These are excluded from all active user interfaces but retained in the database for historical reference (9 existing uploads reference them).
